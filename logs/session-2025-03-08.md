## Critical Finding: Kimi API Key Compatibility

### The Issue
Keys from `kimi.com/code/console` (format: `sk-kimi-...`) are **IDE/coding tool only** keys. They work with:
- Claude Code CLI
- VS Code extensions
- Kimi Code official tools

They do **NOT** work with public Moonshot API endpoints.

### Curl Test Results
```bash
# Test 1: Anthropic format endpoint
curl -s -X POST "https://api.moonshot.cn/anthropic/v1/messages" \
  -H "Authorization: Bearer sk-kimi-..." \
  -d '{"model":"moonshot-v1-8k","max_tokens":50,"messages":[{"role":"user","content":"say hi"}]}'

Result: {"error":{"message":"Invalid Authentication","type":"invalid_authentication_error"}}

# Test 2: OpenAI format endpoint  
curl -s -X POST "https://api.moonshot.cn/v1/chat/completions" \
  -H "Authorization: Bearer sk-kimi-..." \
  -d '{"model":"moonshot-v1-8k","max_tokens":50,"messages":[{"role":"user","content":"say hi"}]}'

Result: {"error":{"message":"Invalid Authentication","type":"invalid_authentication_error"}}
```

### Solution for Online Mode
To use online mode with Kimi, user needs a **Moonshot API key** from https://platform.moonshot.cn (starts with `sk-`, NOT `sk-kimi-`).

**Alternative**: Use DeepSeek (https://platform.deepseek.com) which has free tier and works immediately.
